## 下午/晚上：Phase 3 SoC 集成验证完成

### 根因链（按发现顺序）

1. **Bug-14**：`soc_mem.v` 同步读（`always @(posedge clk) rdata <= mem[...]`）  
   PicoRV32 每条指令读到上一周期的 stale 数据，导致寄存器值全乱  
   → 修复：改为 `assign rdata = mem[addr]`（异步）

2. **Bug-15**：`dram_model.v` DRAM CPU 读端口同样是同步读  
   → 修复：改为 `assign cpu_rdata = mem[cpu_addr[ADDR_W+1:2]]`

3. **Bug-16**：`soc_top.v` SRAM addr 端口位宽 `.addr(mem_addr[21:2])`（20-bit）与实际 22-bit 不符  
   → 修复：改为 `.addr(mem_addr[23:2])`

4. **Bug-17**：固件标记地址 `0x0F00` 在 SRAM 空间（< 0x1000），testbench 监视 DRAM 的 mem[960]  
   → 修复：改标记地址为 `0x2000`（DRAM 空间）

5. **Bug-18**：`$readmemh("soc_test.hex", ...)` 从 `sim/` 工作目录解析路径，加载 10KB 旧 hex  
   → 修复：改为 `../tb/soc_test.hex`

- CPU 正确验证矩阵乘法并写入 PASS 标记到 DRAM 0x2000

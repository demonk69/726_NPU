# CPU Firmware Changes for Multi-Core NPU

## Overview

The firmware (RISC-V assembly, generated by `tools/pth/gen_vgg_closed_loop.py`)
must be modified to:

1. Launch multiple NPU cores concurrently
2. Poll for per-core completion
3. Post-process results from each core
4. Synchronize at layer boundaries

## 1. Current Firmware Structure

From `gen_vgg_closed_loop.py`, the key RISC-V assembly generation functions:

### Per-layer Conv loop (lines 469-590)

```
for m in [0..M step TR]:
    pack A_WORK for this spatial block
    for n in [0..N step TC]:
        configure NPU registers (M, N, K, W/A/R/BIAS addrs, shape)
        NPU start
        wait for NPU done (poll STATUS)
        read R_WORK results
        ReLU + Q24 requantize
        scatter to dense OFM
```

### Current register usage (approximate)

| Reg | Purpose |
|-----|---------|
| s0  | CONST: NPU_BASE |
| s1  | CONST: various |
| s2  | Loop counters |
| s3  | Address bases |
| s4-s11 | Per-layer variables |

## 2. Target Multi-Core Firmware Structure

### Core abstraction: per-core MMIO base

```python
NPU_BASE      = 0x02000000   # Core 0
NPU_CORE_STRIDE = 0x100      # bytes per core
# Core N base = NPU_BASE + N * NPU_CORE_STRIDE
```

### Per-core register offset constants (unchanged from single-core)

```python
REG_CTRL      = 0x00
REG_STATUS    = 0x04
REG_M_DIM     = 0x10
REG_N_DIM     = 0x14
REG_K_DIM     = 0x18
REG_W_ADDR    = 0x20
REG_A_ADDR    = 0x24
REG_R_ADDR    = 0x28
REG_ARR_CFG   = 0x30
REG_CFG_SHAPE = 0x3C
REG_BIAS_ADDR = 0x98
REG_QUANT_CFG = 0x9C
```

### Per-layer multi-core loop (pseudocode)

```
# --- Layer initialization ---
clear entire OFM buffer
ifm_start = current IFM base

for m in [0..M step TR]:
    # --- Pack A_WORK per core ---
    # Each core packs tiles for its own N range but shares the same M block
    for core in [0..NUM_CORES-1]:
        n_start_core = core * N_PER_CORE       # e.g., core0: N=0, core1: N=64
        n_this_core  = (core == NUM_CORES-1) ? (N - n_start_core) : N_PER_CORE
        pack_A_WORK(m, core, ...) → A_WORK_core[core]

    # --- Launch all cores in parallel ---
    for core in [0..NUM_CORES-1]:
        configure_core_regs(core, m, n_start[core], n_count[core])
        write_reg(core, REG_CTRL, CTRL_START)

    # --- Wait for all cores, post-process results ---
    done_mask = 0
    while done_mask != ((1 << NUM_CORES) - 1):
        for core in [0..NUM_CORES-1]:
            if not (done_mask & (1 << core)):
                status = read_reg(core, REG_STATUS)
                if status & STATUS_DONE:
                    # Read R_WORK results, requantize, scatter to OFM
                    postprocess_and_scatter(core, m, n_start[core], n_count[core])
                    done_mask |= (1 << core)

# --- Layer barrier complete ---
# Swap IFM/OFM buffers for next layer
```

### Key differences from single-core

| Aspect | Single-Core | Multi-Core |
|--------|-------------|------------|
| NPU base address | Constant `s0` | Array `s0[core]` = `NPU_BASE + core*0x100` |
| A_WORK buffer | One buffer | Per-core buffers `A_WORK[core]` |
| R_WORK buffer | One buffer | Per-core buffers `R_WORK[core]` |
| N loop | Linear `n += TC` | Per-core base `n = core * N_PER_CORE` |
| Completion wait | Poll one STATUS | Poll all STATUS, track done_mask |
| Scatter destination | `ofm_base + n * ...` | `ofm_base + (core*N_per_core + n) * ...` |

## 3. Per-Core A Tile Packing

The A tile packing function (`pack_a_for_conv` equivalent) must be extended
to produce tiles for each core's N range.

### Weight access remains the same

Each core reads the **same weight tile stream** but at different N offsets.
Since N is the column dimension:

```
W_tile for core C: W[k_base : k_base + K_TILE, n_core_base : n_core_base + TC]
```

The DMA load address for weights:
```
w_load_addr = w_base + n_core_base * K_full_bytes + k_tile_idx * TC * sizeof(INT8)
```

This is computed by the firmware per-core before launching.

### Activation access — shared

All cores for the same M block read the **same A tile stream**:
```
A_tile: A_im2col[m_base : m_base + TR, K_full]
```

The A load address is only a function of `m_base`, not N. So all cores
use the same `a_load_addr` per M iteration.

## 4. Post-Processing Changes

### Current post-processing (single-core)

```python
for row in range(TR):
    for col in range(TC):
        val = read_R_WORK(row, col)           # INT32 raw result
        if val < 0: val = 0                   # ReLU
        val = (val * Q24_mult[n + col]) >> 24 # Q24 requant
        scatter_to_OFM(m + row, n + col, val)
```

### Multi-core post-processing

```python
for core in [0..NUM_CORES-1]:
    n_base_core = core * N_PER_CORE
    n_core_tot  = n_count[core]
    for row in range(TR):
        for col in range(n_core_tot):         # col limited to this core's N range
            val = read_R_WORK_core(core, row, col)
            if val < 0: val = 0
            val = (val * Q24_mult[n_base_core + col]) >> 24
            scatter_to_OFM(m + row, n_base_core + col, val)
```

The scatter destination is naturally disjoint across cores since n_base_core
is different for each.

## 5. Layer Synchronization (Barrier)

```
# After M loop completes, before starting next layer:

# All cores should already be done (last M iteration polled all)
# Just verify and swap buffers
swap IFM ↔ OFM pointers
clear new OFM
```

If any core errors, firmware writes `0xFF` to marker and halts.

## 6. Python Generator Changes (`gen_vgg_closed_loop.py`)

### New parameters

```python
NUM_CORES = 2              # configurable

# Per-core DRAM layout (from dram_layout.md)
A_WORK_BASE = [            # per-core A tile workspace
    0x00050000,            # Core 0
    0x00058000,            # Core 1 (offset by 32KB)
    # ... extend for more cores
]
R_WORK_BASE = [            # per-core result workspace
    0x00068000,            # Core 0
    0x0006C000,            # Core 1 (offset by 16KB)
    # ... extend for more cores
]
```

### Functions to modify

| Function | Change |
|----------|--------|
| `emit_conv_layer_loop()` | Major rewrite: generate multi-core launch + poll loops |
| `emit_npu_launch()` | Add core_id parameter for base address selection |
| `emit_wait_npu_done()` | Rewrite: poll multiple STATUS registers |
| `emit_postprocess_and_scatter()` | Add per-core n_base offset |
| `emit_pack_a_tile()` | Add per-core A_WORK buffer selection |
| Layer initialization | Allocate per-core buffer addresses |

### Firmware size impact

- Additional per-core loop unrolling → ~2x firmware size for 2 cores
- If firmware exceeds SRAM (4KB), may need larger SRAM or compressed loops

## 7. Error Handling

| Error | Detection | Action |
|-------|-----------|--------|
| Core N reports ERR_STATUS != 0 | Check STATUS after each launch | Write 0xFF to MARKER, halt |
| Timeout waiting for core | Cycle counter check | Write 0xFF to MARKER, halt |
| N range misalignment | Pre-launch check | Assert in Python generator |
